# v1.38.3: Drop Embedded Commits from chatroom_workspaceGitState

## Overview

In v1.38.3, the `recentCommits` and `hasMoreCommits` fields have been removed from the `chatroom_workspaceGitState` write path. Commits are now written exclusively through `upsertRecentCommits` → `chatroom_workspaceRecentCommits`.

## What Changed

| Area | Change |
|------|--------|
| `upsertWorkspaceGitState` | No longer accepts or writes `recentCommits` / `hasMoreCommits` args |
| Daemon heartbeat | Now fires two mutations: `upsertWorkspaceGitState` (git metadata) + `upsertRecentCommits` (commits), each with independent hash dedup |
| Schema | Fields kept as `@deprecated optional` for read tolerance; will be removed after migration |
| Frontend | `useLoadMoreCommits` no longer reads `gitState.recentCommits` as a fallback |

## Required Deploy Step

After deploying v1.38.3, run the migration to strip leftover embedded commit data from existing rows:

```bash
cd services/backend
npx convex run migrations:run '{"fn":"migrations:dropEmbeddedRecentCommits"}'
```

Or run all migrations in order:

```bash
cd services/backend
npx convex run migrations:runAll
```

The migration is **idempotent** — safe to re-run. Rows without embedded commits are skipped automatically.

## Schema Cleanup (Future)

Once `dropEmbeddedRecentCommits` has been run against every environment that contains `chatroom_workspaceGitState` rows, the `recentCommits` and `hasMoreCommits` optional fields can be removed from `schema.ts` entirely. This is left for a future PR.
